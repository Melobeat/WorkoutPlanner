# App Icon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the default Android Studio placeholder icon with a bold deep-purple + acid-green outlined dumbbell adaptive icon.

**Architecture:** Two drawable XML files drive the adaptive icon — `ic_launcher_background.xml` (gradient shape) and `ic_launcher_foreground.xml` (vector dumbbell). The `mipmap-anydpi/` XML manifests already reference these correctly and need no changes. Since `minSdk = 33` (Android 13), all devices support adaptive icons; the WebP raster files in `mipmap-*` folders are legacy fallbacks regenerated last.

**Tech Stack:** Android Vector Drawable XML (`<vector>`), Android Shape/Gradient XML (`<shape>`), Android Adaptive Icon API (API 26+)

### Task 1: Replace background with purple gradient

The current `ic_launcher_background.xml` is a `<vector>` with a green fill and a grid overlay (the Android Studio default). Replace it entirely with a `<shape>` gradient. Android's adaptive icon system accepts any drawable type for the background layer.

**Files:**
- Modify: `app/src/main/res/drawable/ic_launcher_background.xml`

- [ ] **Step 1: Replace the file contents**

Write this to `app/src/main/res/drawable/ic_launcher_background.xml` (overwrite entirely):

```xml
<?xml version="1.0" encoding="utf-8"?>
<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="rectangle">
    <gradient
        android:startColor="#1E003E"
        android:endColor="#3D0070"
        android:angle="315" />
</shape>
```

`android:angle="315"` = top-left to bottom-right diagonal (multiples of 45 only; 315 = −45°).

- [ ] **Step 2: Build to confirm no errors**

```bash
JAVA_HOME=/opt/android-studio/jbr ./gradlew assembleDebug 2>&1 | tail -5
```

Expected: `BUILD SUCCESSFUL`

- [ ] **Step 3: Commit**

```bash
git add app/src/main/res/drawable/ic_launcher_background.xml
git commit -m "feat: replace launcher background with purple gradient"
```

### Task 2: Replace foreground with outlined dumbbell vector

The foreground drawable must be 108dp×108dp. The dumbbell artwork is contained within the inner 72dp safe zone (18dp padding on all sides) so no launcher mask shape can clip it. Each element is drawn as a stroke-only path (no fill) in acid green `#C8FF00`.

**Dumbbell geometry on the 108×108 viewport (all within the 20–88 safe range):**

| Part | x | y | w | h | rx |
|---|---|---|---|---|---|
| Left weight plate | 20 | 45 | 14 | 18 | 4 |
| Left collar | 34 | 37 | 8 | 34 | 3 |
| Bar | 42 | 49 | 24 | 10 | 3 |
| Right collar | 66 | 37 | 8 | 34 | 3 |
| Right weight plate | 74 | 45 | 14 | 18 | 4 |

### Task 3: Regenerate WebP raster fallbacks

The `mipmap-{mdpi,hdpi,xhdpi,xxhdpi,xxxhdpi}/` folders contain WebP raster bitmaps — legacy fallbacks for devices that predate adaptive icons (API < 26). Since `minSdk = 33`, these will never be used at runtime. However, regenerating them keeps the project consistent and avoids the default Android robot showing in any tooling that reads the raster files.

Use **Android Studio's Image Asset Wizard** — the standard, no-dependency approach:

**Files:**
- Replace: all 10 WebP files across `mipmap-{mdpi,hdpi,xhdpi,xxhdpi,xxxhdpi}/`

- [ ] **Step 1: Open Image Asset Wizard**

In Android Studio: Right-click `app/src/main/res` → **New → Image Asset**.

- [ ] **Step 2: Configure as Adaptive Icon**

  - **Icon type:** Launcher Icons (Adaptive and Legacy)
  - **Foreground layer:** choose **Source Asset → Resource** → pick `@drawable/ic_launcher_foreground`
  - **Background layer:** choose **Source Asset → Resource** → pick `@drawable/ic_launcher_background`
  - **Legacy tab:** leave defaults (generates the WebP fallbacks)
  - Click **Next → Finish**

Android Studio overwrites all `mipmap-*/ic_launcher.webp` and `ic_launcher_round.webp` files.

- [ ] **Step 3: Build to confirm**

```bash
JAVA_HOME=/opt/android-studio/jbr ./gradlew assembleDebug 2>&1 | tail -5
```

Expected: `BUILD SUCCESSFUL`

- [ ] **Step 4: Commit**

```bash
git add app/src/main/res/mipmap-hdpi/ app/src/main/res/mipmap-mdpi/ \
        app/src/main/res/mipmap-xhdpi/ app/src/main/res/mipmap-xxhdpi/ \
        app/src/main/res/mipmap-xxxhdpi/
git commit -m "feat: regenerate launcher WebP raster fallbacks for new icon"
```
